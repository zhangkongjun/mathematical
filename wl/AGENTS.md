# AGENTS.md

## 适用范围
本文件只补充 `wl/` 目录规则；通用编码、验证和任务目录总约定仍以上级 `AGENTS.md` 为准。

## 目录职责
- 本目录存放 Mathematica / Wolfram 核心代码。
- 本目录中的源码与包文件默认使用 `UTF-8` 编码。

## 使用规则
- 公共可复用逻辑放在 `wl/common/`。
- 任务专属实现优先放在 `wl/tasks/<task-slug>/`，验证可复用后再上收。
- 接口应显式传入输入路径、关键参数和输出位置。
- 改动后优先通过最小 `wolframscript` 或默认脚本入口验证。
- 对于 Codex 生成且已验证通过的最终 `.wl` / `.wls`，补充详细的简体中文注释。
- 注释至少说明：代码做什么；使用或维护时要注意什么；当前实现的限制或适用边界；当前使用到的Wolfram 功能和函数，以及这些Wolfram功能与函数起到的作用。

## 代码格式规则
- `wl/` 目录下的 `.wl`、`.wls` 默认采用“紧凑但分层清晰”的排版风格，目标是减少无意义空行，同时保留足够的结构感，方便人直接阅读和 diff。
- 不要在同一段连续代码块内部插入无作用的空白行；只有在“定义与定义之间”“注释块与后续定义之间”“明显不同语义层之间”才保留单个空行。
- 顶层定义之间默认使用一个空行分隔；不要连续保留两个及以上空行。
- 短小、语义单一的表达式可以保持单行；一旦单行会明显过长、包含多个键值对、多个选项或多层嵌套，就应主动拆成多行。
- `ClearAll[...]`、函数参数列表、长选项列表、长字符串映射表等，优先使用每项一行的垂直排版。
- `Association` 若包含多个键，默认写成多行形式：
  - 每个键值对单独一行；
  - 结尾的 `|>` 单独占一行；
  - 同一层级的多个 `Association` 保持一致格式。
- `List`、规则列表、选项列表在元素较多时应按“每项一行”排版；若只是两个很短的元素，可保留单行。
- `Module`、`With`、`Table`、`Manipulate`、`Graphics` 等多层结构应通过稳定缩进展示层级，不要为了“对齐好看”引入额外空行。
- 在 `If`、`Which`、`Table`、`Join`、`Graphics` 等内部，只有当空行能明确隔开两个不同语义段时才保留；不要在每个子表达式之间机械插空行。
- 注释块保持完整、连续，注释内部允许按“代码作用 / 使用注意 / 当前边界 / Wolfram 功能”分行，但注释块与其对应定义之间不要再插额外空行。
- 结构化说明源文件中的 `(* ::Title:: *)`、`(* ::Section:: *)`、`(* ::Text:: *)` 单元标记继续按 notebook 源文件习惯保留，不按普通代码块压缩成过度紧凑的样式，以免影响说明层维护。
- 当代码已经采用某种一致格式时，后续修改应尽量局部延续原风格；若确认现有风格明显降低可读性，则按本节规则整体收敛，不要混用两套排版方式。

## 子目录说明
- `wl/common/`：跨任务复用的公共函数。
- `wl/data/`：数据读取、清洗、转换与导出入口。
- `wl/ai/`：可复用的模型调用、特征处理、评估或实验辅助逻辑。
- `wl/astronomy/`：明确单位、时间系统和数据约定的代码。
- `wl/physics/`：明确参数、边界条件、初值和数值设置的代码。
- `wl/probability/`：显式暴露随机种子和样本规模处理的代码。
- `wl/weather/`：明确站点、时间范围、采样粒度和缺失值策略的代码。
