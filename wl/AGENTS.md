# AGENTS.md

## 目录职责
本目录存放 Mathematica/Wolfram 语言核心代码。

## 使用规则
- 公共与可复用逻辑可放在这里统一维护，但 notebook 展示或直接引用的核心逻辑以 notebook 单元格中的版本为准。
- 新增 `.wl` 或 `.wls` 代码时，按领域子目录归类，并与 notebook 中对应单元格保持一致。
- 保持函数接口清晰，显式传入输入路径、参数和输出位置，避免硬编码环境依赖。
- 变更后优先通过最小 `wolframscript` 或脚本入口做验证。
- 新研究任务的专属 Wolfram 代码优先写到 `wl/tasks/<task-slug>/`；只有稳定复用后才上收至公共或领域目录。
- 针对 Codex 生成且已经自动测试通过的最终 `.wl` 与 `.wls` 代码，默认添加简体中文注释。
- 中文注释至少应说明三类信息：代码功能、使用或维护时的注意事项、当前实现的限制或适用边界。
- 注释优先写在文件头、主要函数定义前、关键数据流或关键数值处理代码块前，不要只在文件末尾补一句笼统说明。
- 若某段逻辑已经足够直观，可减少重复注释；但最终交付代码中仍应保留对核心函数和关键约束的中文说明。

## 子目录约定
- `wl/tasks/` 用于存放任务级 Wolfram 代码，每个任务一个子目录，避免不同课题直接共用同一套未稳定的实现与设置。
- `wl/common/` 只放跨领域复用的公共函数，避免混入领域专属逻辑。
- `wl/data/` 负责数据读取、清洗、转换和导出入口，保持对 `data/raw/` 只读，派生结果写到 `data/processed/`。
- `wl/ai/` 放可复用的模型调用、特征处理、评估或实验辅助函数，随机相关逻辑显式暴露随机种子或随机源设置。
- `wl/astronomy/` 明确单位、时间系统、输入数据格式以及导出路径约定。
- `wl/physics/` 明确参数含义、单位、边界条件、初值以及数值实验关键参数。
- `wl/probability/` 实现分布、采样、估计和统计检验等核心逻辑，并允许外部传入随机种子。
- `wl/weather/` 明确站点、时间范围、采样粒度和缺失值处理策略，外部数据输入路径保持参数化。
