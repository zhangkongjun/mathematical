# AGENTS.md

## 目录职责
本目录存放 Codex 运行配置与代理定义的入口文件。

## 使用规则
- 先遵循上级目录的 `AGENTS.md`，这里仅补充本目录约束。
- 优先维护 `.toml` 配置，不要把代理约定散落到其他临时文件。
- 修改配置前先读取已有字段，尽量延续现有键名、缩进和注释风格。
- 不在这里保存实验结果、原始数据或 notebook 导出物。
- 若配置引用仓库内路径，使用相对路径并确保路径在仓库中真实存在。
- 涉及敏感信息时保留模板占位，不要写入真实凭据。
- 主 agent 的配置应优先保证“完成即退出”，不要把空闲等待当作默认行为。
- 若任务过程中拉起子代理、内核、外部解释器或后台命令，主 agent 在给出最终结果前负责清理并退出，避免继续占用 CPU、GPU、内存与文件句柄。
- 只有在用户明确要求持续运行、守护、定时或人工后续接管时，才允许保留会话或后台进程。

## 子目录约定
- `.codex/agents/` 中的代理定义沿用本目录规则，不再单独维护重复模板。
- 代理定义保持单一职责，优先写成声明式配置，并与仓库中的 `wl/`、`scripts/`、`notebooks/` 实际流程保持一致。
- 代理说明中若出现输入、输出或日志路径，统一使用仓库内已约定的目录。
- `.codex/wolfram-envs/` 用于存放课题级 Wolfram 环境设置；每个任务一个子目录，不要让多个研究任务共享同一份环境配置。
- 若任务需要环境说明、启动包装脚本或初始化文件，优先放到 `.codex/wolfram-envs/<task-slug>/`，不要写进全局用户环境。
