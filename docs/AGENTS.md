# AGENTS.md

## 适用范围
本文件只补充 `docs/` 目录规则；通用流程、编码、语言与任务文档总要求仍以上级 `AGENTS.md` 为准。

## 目录职责
- `docs/` 只保存规则文档、任务说明、决策记录、迭代记录与最终文件索引。
- 本目录中的文本文件默认使用 `UTF-8` 编码。

## 强制规则
- `docs/` 目录只保存 Markdown 文档，不保存可执行代码，不保存 `.wl`、`.wls`、`.ps1`、`.nb` 等实现文件。
- 需要修改 notebook 行为时，应修改 `wl/` 或 `scripts/` 中的源实现，不要把实现逻辑写进 `docs/`。
- 需要修改 notebook 内容时，应回到源文件或导出链路中修改，不要把“最终内容”先写到 `docs/` 再人工搬运到 notebook。
- `docs/` 中可以说明代码路径、命令模板、目录约定和生成规则，但这些说明只是文档，不是实现本体。
- 任何稳定下来的规则、流程和目录约束，都必须回写到 `docs/`，不能只留在聊天记录里。

## 任务文档要求
- `docs/tasks/<task-slug>/FINAL_FILES.md` 用于给用户与 Codex 提供最终交付入口，必须明确推荐查看的最终文件、推荐命令，以及最值得回溯的过程文件与目录。
- `FINAL_FILES.md` 应显式指向 `DECISIONS.md` 与 `ITERATIONS.md`。
- `docs/tasks/<task-slug>/DECISIONS.md` 必须写成正式的研究与实现决策说明，目标不仅是记录“为什么这样实现”，还要作为后续学习、研究复盘以及与网页版 ChatGPT 持续沟通的背景材料。
- `DECISIONS.md` 默认至少应包含：课题的科学背景与问题定义；相关的数学知识、物理理论与核心概念；涉及公式时的详细解释、符号说明、适用条件与尽量标准的数学公式表达；当前课题研究现状、已知限制与遗留问题；未来可能的发展方向、可继续验证的假设与可扩展实现路径。
- `DECISIONS.md` 的写法应尽量自洽、可独立阅读，避免只写结论不写背景；应让不了解上下文的人也能借助该文档快速补齐研究背景，并据此继续开展实现、分析或讨论。
- 若任务经历多轮迭代，默认维护 `docs/tasks/<task-slug>/ITERATIONS.md`，按轮次记录目标、优化点、结果、实现路径、过程文件、日志和失败尝试。
