# 八大行星轨道演示 决策说明

## 1. 实现目标与适用范围
1. 本任务的直接目标，是在仓库内重建一个“八大行星运行轨迹动画”的 Wolfram 版本，并同时生成便于阅读、调试和扩展的 `.nb` 文件。
2. 当前实现优先服务于教学演示、可视化讲解、交互式 notebook 调试，以及后续在本仓库中继续维护。
3. 当前实现不是高精度星历计算器，不适合作为观测级预测或严肃科研结论的直接依据。

## 2. 当前稳定方案的核心决策
1. 最终实现收敛为单个 `.wl` 源文件，并在该文件内同时保留结构化说明单元标记与最终可执行代码。
2. 最终 notebook 通过仓库级 `scripts/export_nb.ps1 -GenerationMode StructuredSource` 生成。
3. 不再采用“先生成代码 notebook，再重写 `.nb` 文本”的旧链路，因为那条链路会在当前环境下破坏中文说明。
4. 该方法后续已上收为仓库级 `StructuredSource` 能力。
5. 若需要完整回溯每轮迭代、失败尝试与过程文件，请优先查看：
   `D:\WorkCode\mathematical\mathematical\docs\tasks\eight-planets-orbit-demo\ITERATIONS.md`

## 3. 参考来源与工程选择
1. 参考了 `D:\WorkCode\wolfram\generate_planet_demo_via_server.wl` 的 notebook 生成思路。
2. 参考了 `D:\WorkCode\wolfram\EightPlanetsOrbitDemo_server.nb` 的动画结构与显示组织。
3. 关键工程选择是“说明与代码同源”，而不是“说明与代码分离后再拼到 `.nb` 里”。

## 4. 核心假设
1. 所有行星轨道都按二维平面上的圆轨道处理。
2. 所有行星默认共面。
3. 角速度由平均公转周期直接决定，即假设角速度恒定。
4. 太阳固定在原点。
5. `year` 是抽象的模拟年份，而不是绑定到真实历元的日期。
6. 行星点大小、标签偏移、尾迹长度和可见性增强都优先服务于显示效果。

## 5. 为展示与调试做的优化
1. 将行星颜色、轨道半径、公转周期和标签偏移集中存放在 `planetData` 中，降低调参成本。
2. 对行星显示半径使用温和的对数增强，让外行星在全景视图中仍可辨识。
3. 为每颗行星增加最近一段彩色尾迹，增强运动可读性。
4. 提供多档 `zoom` 预设，方便快速切换内行星视图和全景视图。
5. 使用中文标签、深色背景和高对比前景色，优先优化 notebook 阅读体验。
6. 将说明与代码保持在同一个 `.wl` 中，避免 notebook 代码与实际维护代码脱节。

## 6. 已知限制、缺陷与偏差
1. 真实轨道不是完美圆轨道，当前模型忽略了偏心率影响。
2. 当前是二维共面模型，掩盖了真实三维轨道结构。
3. 初始相位没有对齐到任何真实历元。
4. 常量角速度省略了近日点和远日点速度变化。
5. 行星显示尺寸经过人为增强，不符合真实比例。
6. 当前视图是理想化日心平面视图，不是观测者视角。
7. 缩放、标签偏移或字体环境变化时，标签仍可能重叠。
8. 当前模型未包含摄动、共振、卫星系统等更复杂动力学因素。

## 7. 运行时风险
1. 最终 notebook 生成依赖 Wolfram Front End；若 Front End 启动失败或状态异常，生成会失败。
2. 当前标签布局偏向 Windows 中文字体环境，在其他机器上可能发生布局变化。
3. `Manipulate` 的流畅度依赖机器性能、Front End 状态和图形复杂度。
4. 若扩展 `zoom` 但不联动调整布局参数，可读性会下降。
5. 当前最终 notebook 默认保留结构化说明单元和一个完整 `Input` 单元，不会自动附带预计算输出。

## 8. 可优化方向
1. 用椭圆轨道参数与开普勒方程替代当前圆轨道近似。
2. 加入轨道倾角、升交点经度和近心点角等三维轨道要素。
3. 将初始相位绑定到明确历元。
4. 接入 `AstronomicalData`、JPL DE 或其他更高精度数据源。
5. 将轨道参数与显示参数拆分为独立数据文件。
6. 增加速度矢量、近日点/远日点标记和双显示模式。
7. 增加批量导出 PNG / GIF / 视频的任务脚本。

## 9. 相关课题与研究点
1. 开普勒定律的可视化教学表达。
2. 近似太阳系轨道模型与高精度模型的对比。
3. 不同参考系下的太阳系运动展示。
4. 会合、冲日、逆行等天象事件的可视化检测。
5. “源代码即展示材料”的科学 notebook 设计方式。
6. 多尺度科学可视化中可读性与真实性的平衡。

## 10. 维护建议
1. 若后续目标仍以教学演示为主，应继续保留当前近似模型，但始终明确适用范围与限制。

3. 若后续目标转向代码复用，应进一步拆分参数层、动力学层、渲染层和 notebook 导出层。
4. 若 notebook 生成规则继续迭代，应同步维护 `DECISIONS.md`、`ITERATIONS.md` 与 `FINAL_FILES.md`。
