# AGENTS.md

## 适用范围
本文件只补充 `notebooks/` 目录规则；命令模板与 `export_nb.ps1` 模式选择以 `scripts/AGENTS.md` 为准。

## 目录职责
- 本目录用于保存 notebook 源文件、任务 notebook 与导出前后的 notebook 资产。
- 本目录中的文本源文件默认使用 `UTF-8` 编码。

## 强制规则
- 不要直接修改最终 `.nb` 文件。
- 生成、更新或修复 `.nb` 时，优先使用 `scripts/export_nb.ps1`。
- 若 notebook 说明文档有变化，应回到 `.wl` / `.wls` 源文件或 `export_nb.ps1` 生成链路中修改。
- 只有仓库默认入口确实无法覆盖当前需求时，才新增任务级 notebook 生成脚本。
- 若某种方法已经证明可复用，应优先上收到仓库级入口，而不是长期保留重复的任务级实现。

## notebook 内容组织
- 若 notebook 同时承担阅读入口、讲解入口或调试入口，应保留结构化说明单元，而不是只导出裸代码。
- 若 notebook 由源码自动生成，说明文档默认先写入 `.wl` 或 `.wls`，再生成 `.nb`。
- 若需要程序化写入 notebook 单元，直接使用 `NotebookWrite`，并优先配合 `ToBoxes`。
- 涉及公式时，优先使用标准数学表达，再补充其与代码参数或函数的对应关系。

## 结构化说明的默认骨架
- 标题。
- 目标与适用范围。
- 核心假设。
- 主要数学概念与物理公式，或相关科学理论与公式。
- 为展示、调试或教学做的优化。
- 已知限制、偏差与运行注意事项。
- 后续研究或扩展方向。
- 最终可执行代码。
